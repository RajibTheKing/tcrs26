\section{Introduction}
\label{sec:intro}
    The rapid technological advancement in railway systems is driving a fundamental shift from manual human control to autonomous operations. This transformation is particularly evident in the pursuit of higher grades of automation, i.e. GoA levels 3 and 4, where trains operate with minimal human intervention or become completely driverless. As railway systems progress toward full automation, the role of remote control systems becomes increasingly critical. The Remote control system is a vital layer in railway operations, providing essential fallback capabilities when autonomous systems encounter failures \cite{Shift2Rail2019, CoganMilius2023, 10.1145/3696355.3699698}.

    To fulfill this critical fallback role effectively, a remote control system must support two fundamental communication channels: one for transmitting control commands and another for streaming real-time video. In most existing teleoperated systems, control commands are typically transmitted using reliable protocols such as MQTT or WebSocket over TCP, while video streaming is handled by UDP-based protocols like RTSP or WebRTC \cite{10597107, 11416303, fezeu2025teleoperatingautonomousvehiclescommercial}. However, QUIC protocol, standardized as the foundation for HTTP/3, has emerged as a promising alternative to traditional TCP- and UDP-based communication for Time centric reactive software systems \cite{10.1145/3696355.3699698}. Built based on QUIC, WebTransport offers two key features: reliable streams suitable for control command transmission, and unreliable datagrams optimized for low-latency video streaming, can be used in both communication channels with reduced architectural complexities \cite{10.1145/3696355.3699698, mejías2025streamingremoterenderingservices}.

    WebTransport is a web-based protocol primarily constrained to standard web browser environments. However, standard web browsers does not provide guarantees required for safety-critical system. In our case the web browser is isolated exclusively to non-safety-critical tasks, specifically video stream visualization and control command manual inputs. The underlying safety-critical operational logic is decoupled from the user interface and executed directly on the train client side. To enforce deterministic execution within this cyber-physical domain, the train client leverages Lingua Franca, a reactor-oriented coordination language which guarantees determinism by construction by decoupling logical time from physical wall-clock time. This approach eliminates race conditions and non-deterministic callback ordering of traditional publish-subscribe patterns, ensuring predictable execution traces on the physical train system \cite{teper2026deterministicexecutionros2}.
    
    This paper provides following contribution:
        \begin{itemize}
            \item A comprehensive architectural design of a train client using Lingua Franca (LF), an important component of remote control systems.
            
            \item Provide a performance analysis of WebTransport in the context of remote control system under realistic railway network conditions.
        \end{itemize}
