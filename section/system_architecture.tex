	Figure \ref{fig:overview} illustrates the high-level structure of our system, highlighting the main modules and their interconnections. The \textsf{TrainClient} runs on the train, collects sensor data from the physical world and reacts upon receiving control commands. The \textsf{RemoteControlClient} is a web application that provides a user interface for operators, rendering a real-time video stream and sending control commands. The \textsf{CentralServer} acts as a communication hub between the \textsf{TrainClient} and the \textsf{RemoteControlClient}, relaying information between them. In this paper, our focus is on the architecture of the \textsf{TrainClient}, which is implemented using Lingua Franca.

	In Lingua Franca, a \emph{reactor} is a concurrent, stateful component with well-defined input and output ports, timers, and actions. Reactions are functions triggered by events, execute atomically and deterministically. Lingua Franca enforces a logical time semantics: all events are scheduled and processed in timestamp order, and physical time can be explicitly referenced for real-time interactions. Figure~\ref{fig:train-client} illustrates the top-level components of the \textsf{TrainClient}. The diagrams are automatically generated using the diagramming framework KLighD\footnote{https://github.com/kieler/KLighD} and the Eclipse Layout Kernel (ELK)~\cite{KasperowskiDvH24}.

	The \textsf{TrainClient} is the main \emph{reactor} containing all other \emph{reactors}. The \textsf{VideoStreamer} is responsible for capturing and encoding video frames from the physical camera mounted on the train. The encoded video frames are then passed to both \textsf{NetworkProtocol} and \textsf{DataStorage}. The \textsf{DataStorage} is responsible for storing the video frames and other relevant data locally on the train. The \textsf{NetworkProtocol} handles the connection establishment and data transmission channel with the \textsf{CentralServer}. Upon data reception, the \textsf{NetworkProtocol} forwards the received data to the \textsf{Parser}. When the \textsf{Parser} identifies a new connection request, it sends output signals to initialize the \textsf{ClockSynchronizer} and the \textsf{HeartbeatHandler}. The \textsf{ClockSynchronizer} is responsible for calculating the clock offset between the train and the remote control and the \textsf{HeartbeatHandler} manages the periodic exchange of heartbeat messages to monitor active connectivity. Upon receiving control commands, the \textsf{Parser} forwards them to the \textsf{Controller} which processes the commands and translates them into actionable tasks for the physical world and actuates the train accordingly.

	The \textsf{HeartbeatHandler}, shown in Figure \ref{fig:heartbeat}, demonstrates how Lingua Franca's logical-time semantics and \emph{modal reactors} can be used to implement a robust connectivity monitoring mechanism. With modal models, \emph{reactors} enable the coordination of reactive tasks based on modes of operation~\cite{SchulzRosengartenvHLB+23c}. The \textsf{HeartbeatHandler} operates in three modes: \textsf{Init}, \textsf{Send-and-Check}, and \textsf{Stopped}. If a heartbeat initialization signal is received, the \emph{reactor} switches the mode from \textsf{Init} to \textsf{Send-and-Check} and schedules a logical action with delay of 3000~ms. Within this time window atleast one heartbeat message is expected to be received from the remote control. The \emph{reactor} uses a timer with a period of 500~ms to transmit heartbeat messages. It reschedules the logical action upon receiving a new heartbeat message from the remote control. If the heartbeats stop signal is received or the timeout expires, a timeout signal is sent and the \emph{reactor} switches to \textsf{Stopped}, indicating loss of connectivity or inactive remote control.

	% Upon receipt of a new heartbeat message, it reschedules the logical action. If a heartbeat stop signal is received, or the delay window expires without receiving a new heartbeat message, the reactor sends a heartbeat timeout signal and transitions to the \textsf{Stopped} mode, indicating either a loss of connectivity or the absence of an active remote control.
