\section{System Architecture}
\label{sec:system-architecture}

  \subsection{System Overview}
    This subsection presents an overview of the system architecture, emphasizing its core components and the interactions among them. Figure~\ref{fig:system-overview} depicts the high-level structure of the system, illustrating the main modules and their interconnections. The architecture is designed to support multiple trains, each of which connects to a centralized server hosted on a cloud platform via a network connection. Every train is equipped with a set of onboard sensors and communication devices that enable continuous data acquisition and seamless transmission to the central server. On the other side, multiple remote control interfaces—accessible through any standard web browser—can establish connections with the central server for monitoring and control purposes. Through these interfaces, an operator can select any train currently connected to the server, observe its real-time operational data, and issue control commands. This setup facilitates efficient, scalable, and real-time management of train operations across distributed environments.
