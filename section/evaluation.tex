\section{Experiment and Evaluation}
\label{sec:evaluation}
    We focused on validating the proposed remote train control architecture by evaluating the end-to-end latency of control commands and video streams under realistic network conditions. The experiments were conducted on the REAKT\footnote{https://www.reakt.sh} test track in Northern Germany between Malente and Lütjenburg, which serves as a real-world laboratory for innovative rail technologies. The \textsf{TrainClient} was deployed on a Raspberry Pi 5, which coordinates the test vehicle equipped with a camera and a motor controller. The \textsf{CentralServer} was deployed on a cloud instance hosted in Frankfurt, Germany. During the experiments, the test vehicle in Lütjenburg and the remote control setup at Kiel University were both located approximately 480 km from the server, while the distance between the test vehicle and the remote control setup was approximately 31 km. In these experiments, the test vehicle was connected to the server over QUIC protocol via 5G network using a Chateau 5G R17 AX router, while the remote control was connected to the server over WebTransport protocol using a wired Ethernet cable. Figure \ref{fig:experiment_setup} shows real-time remote control setup and the test vehicle on the track.

    Figure \ref{fig:latency_measurements} presents the end-to-end latency measurements for control commands and the rolling average latency for video frames, calculated with a window size of 30 frames. The results indicate that all control commands were received within acceptable limits. Additionally, no control commands were lost during the trip sessions. Consequently, the reliable channel enabled by the QUIC-based WebTransport protocol is suitable for transmitting control commands in remote train control systems.

    We received 17,387 video frames and lost 44 during the session, which is expected under real-world network conditions. Average latency stayed within acceptable limits, with a few spikes above the threshold. If such spikes become frequent or prolonged, the remote control interface should alert the operator and take safety actions such as reducing speed.
